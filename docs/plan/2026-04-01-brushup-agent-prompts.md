# AI エージェント全体ブラッシュアップのプラン

## Context

bioRxiv 論文リサーチエージェントで 2 つのクエリ（Alternative Polyadenylation / 生成AI×ゲノム解析）を実行した結果、以下の状態が確認された:

**良くなった点:**
- DOI 重複排除が機能し、サブタスク間で異なる論文がヒット
- サブタスク分解がメタタスクではなく具体的な調査内容になった
- 論文フィルタリング: 15 件中 14 件が関連あり（除外 1 件のみ）

**まだ改善が必要な点:**
- 関連度スコアが全体的に中程度（0.44〜0.56）
- タスク評価プロンプトが曖昧で、情報の十分性を適切に判定できない
- クエリ拡張のリトライフィードバックが汎用的すぎて改善につながりにくい
- SimpleAnalyzer の判定基準が「寛容に」だけで具体性に欠ける
- Reporter が引用数に偏重し、矛盾する研究の統合方法が不明確

---

## 改善 1: TaskEvaluator プロンプトの大幅強化 (CRITICAL)

**問題**: 現在のプロンプトはルール 2 つだけで極めて曖昧。LLM が何を基準に「情報不足」と判断すべきか明確でない。

**ファイル**: `packages/@ai-suburi/core/chapter6-biorxiv/chains/prompts/task_evaluator.prompt`

以下の評価基準をプロンプトに追加:

```
## 評価基準

以下の観点で、収集された情報がgoal_settingの達成に十分かどうかを判断してください。

1. **網羅性**: goal_settingで求められている主要なトピックや観点がカバーされているか
2. **具体性**: 各論文の分析結果に具体的な手法名、データセット名、数値結果が含まれているか
3. **多様性**: 異なる角度やアプローチの研究が含まれているか（同じグループの類似研究ばかりではないか）
4. **最新性**: goal_settingで求められている時期の論文が含まれているか

## 判断の目安

- 上記 4 観点のうち 3 つ以上が満たされている → need_more_information: false
- 2 つ以下しか満たされていない → need_more_information: true（content に不足している観点を具体的に記述）
- 論文が 3 件未満 → 原則として need_more_information: true
```

## 改善 2: SimpleAnalyzer プロンプトの具体化 (HIGH)

**問題**: 「寛容に判定してください」だけで、何を基準に関連性を判定するか不明確。

**ファイル**: `packages/@ai-suburi/core/chapter6-biorxiv/chains/prompts/simple_analyze.prompt`

判定基準を具体化:

```
1. この論文が調査タスクに関連しているかどうかを判定してください。
   - 以下のいずれかに該当する場合は関連ありと判断してください:
     a) アブストラクトに、タスクで言及されている技術・手法・概念が直接含まれている
     b) アブストラクトの研究対象が、タスクのテーマと同じ生物学的プロセスや分野に属する
     c) アブストラクトで紹介されているツールや手法が、タスクの目的に応用可能である
   - 以下の場合にのみ関連なしと判断してください:
     a) 論文の研究対象がタスクのテーマと完全に異なる分野である
     b) アブストラクトから得られる情報がタスクに対して全く有用でない
```

回答生成の指示も強化:

```
2. 関連ありの場合は、以下の構成で回答を生成してください:
   - **主要な発見**: アブストラクトから得られる最も重要な知見（1〜2文）
   - **使用された手法**: 論文で使用されている具体的な技術名・ツール名
   - **意義**: この研究がタスクのテーマにどう貢献するか（1文）
```

## 改善 3: クエリ拡張プロンプトの改善 (HIGH)

**問題**: リトライ時のフィードバックが「より一般的なキーワードで」という汎用的な指示だけ。

**ファイル**: `packages/@ai-suburi/core/chapter6-biorxiv/rag/rag-searcher.ts`

### 3a: リトライフィードバックの具体化

現在（L171-172）:
```
feedback = '検索結果が0件または関連度が低い結果のみでした。より一般的なキーワードや別の関連用語で検索してください。';
```

変更後:
```
feedback = `前回のクエリ "${expandedQuery}" では関連度の高い論文が見つかりませんでした。以下の戦略で別のクエリを生成してください:
1. より上位の概念カテゴリで検索する（例: 特定の手法名 → その手法が属する分野名）
2. 同義語や別の表記法を使用する（例: scRNA-seq → single-cell transcriptomics）
3. 関連する生物学的プロセスやパスウェイ名で検索する`;
```

### 3b: リランキングの閾値を緩和

現在: `RELEVANCE_SCORE_THRESHOLD = 0.3`

学術論文では専門用語のバリエーションが大きいため、0.3 は厳しすぎる。`0.25` に緩和する。

## 改善 4: Reporter プロンプトの統合分析指示を強化 (MEDIUM)

**問題**: 引用数に偏重し、研究の統合・矛盾の分析方法が不明確。

**ファイル**: `packages/@ai-suburi/core/chapter6-biorxiv/chains/prompts/reporter_user.prompt`

処理手順の「分析フェーズ」に追加:

```
2. 分析フェーズ
  - 関連する結果を一貫したセクションに整理します。
  - 異なる研究で使用される方法論を比較します。
  - 分析から浮かび上がるパターンを特定します。
  - 見られる矛盾や不一致を文書化します。
  - 研究間の手法・対象データ・結果の違いを比較テーブルで整理します。
  - 同じテーマに対する異なるアプローチがある場合は、それぞれの利点と限界を明示します。
```

評価項目の修正（引用数偏重を是正）:

```
1. 収集された論文が適切にグループ化・統合されていると高評価です。
2. 分析結果が比較テーブルで表現されていると高評価です。
3. 具体的な手法名・ツール名・数値結果が含まれていると高評価です。
4. 論文のタイトルとDOIを明示的に引用していると高評価です（論文IDだけの参照は低評価です）。
5. 研究間の一致点・相違点が明確に述べられていると高評価です。
```

## 改善 5: クエリ拡張プロンプトにドメイン知識を追加 (MEDIUM)

**問題**: クエリ拡張が「1〜3 キーワード」の一般的な指示のみで、バイオインフォマティクス固有の検索戦略がない。

**ファイル**: `packages/@ai-suburi/core/chapter6-biorxiv/rag/rag-searcher.ts` の `EXPAND_QUERY_PROMPT`

ルールに追加:

```
6. バイオインフォマティクス特有の表記揺れを考慮すること:
   - 略語と正式名称の両方を含める（例: APA / Alternative Polyadenylation）
   - ハイフン有無の揺れを考慮する（例: single-cell / single cell）
   - 技術名とその応用分野を組み合わせる（例: "machine learning" + "genomic variant"）
```

## 検証方法

1. 同じ 2 クエリ（Alternative Polyadenylation / 生成AI×ゲノム解析）で再実行
2. 確認ポイント:
   - 関連度スコアが 0.25〜0.56 の範囲でより多くの論文が残るか
   - TaskEvaluator が網羅性・具体性・多様性を基にした具体的な評価を返すか
   - SimpleAnalyzer が「主要な発見」「使用された手法」「意義」の構造化回答を返すか
   - レポートに研究間の比較テーブルが含まれるか
   - リトライ時のクエリが前回と明確に異なるアプローチになるか
